W4U Writing Wizard — App Summary

What It Is

W4U Writing Wizard is an Italian-first AI authoring studio that orchestrates an n8n-driven, multi-agent pipeline plus a Vite wizard UI to turn a shy idea into a complete manuscript.

Who It's For

Aspiring and professional fiction or nonfiction authors who want guided AI scaffolding, structure, and multi-format output while keeping creative control.

What It Does

  • Refines raw input through Interviewer and Concept Gen1 agents that capture blueprint details and surface six structured concept cards for direction.
  • Generates plot outlines and chapter indexes via Plotter and Architect, aligning chapter count with the target page budget.
  • Splits chapters into scenes or sections with the Scaffolder, then tasks the Writer with drafting guided, ~250-word paragraphs using the Blueprint, adjacent context, and RAG vector chunks.
  • Optionally routes output through the Editor agent to surface incoherencies before locking a paragraph.
  • Runs a React wizard made of Blueprint, Concept, Configuration, Production, Scaffold Revision, Cover, and Export views that hook into n8n flows for each stage.
  • Backed by an Express + Supabase backend so users can download PDF, EPUB, or DOCX files while logging exports and rehydrating chapter data.

How It Works

  • The front end (Concept, Configuration, Blueprint pages) triggers n8n actions such as INTERVIEW, GENERATE_CONCEPTS, OUTLINE, and SCAFFOLD_CHAPTER, producing the Blueprint JSON, plot summary, and chapter targets stored in Postgres.
  • Scaffolder records numbered scenes/sections in Postgres while Writer pulls the Blueprint, recent paragraphs, and RAG vector DB chunks to keep tone, then the optional Editor reviews coherence before finalizing each paragraph.
  • An Express server (npm run server) speaks to Supabase with the service-role key, records debug logs, and feeds the ExportPage UI so completed books can be streamed as PDF, EPUB, or DOCX files.

How to Run

  • npm install
  • npm run dev # starts the Vite UI on localhost:5173
  • npm run server # starts the Express backend that uses the .env Supabase / n8n webhook configuration for exports
